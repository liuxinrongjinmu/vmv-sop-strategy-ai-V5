## 一、产品概述

### 1.1 产品定位

VMV-SOP战略咨询系统是一款基于"From VMV to SOP"理论的AI驱动战略咨询工具。系统通过多Agent协作，帮助创业者和企业管理者进行深度战略分析，核心价值在于：

- **降低战略决策风险**：通过正反论据推演，暴露预判中的盲点
- **提升战略思考深度**：通过引导式对话，激发用户思考
- **沉淀战略分析资产**：生成可追溯、可复用的分析报告

### 1.2 目标用户

| 用户画像 | 核心诉求 | 使用场景 |
|---------|---------|---------|
| 创业者（0-1阶段） | 验证赛道预判、明确战略方向 | 融资前、转型期、新业务启动 |
| 企业高管（1-10阶段） | 制定中长期战略、分解执行路径 | 年度战略会、业务规划 |
| 战略咨询顾问 | 提升咨询效率、标准化输出 | 客户服务、方案撰写 |

### 1.3 核心差异化价值

| 维度 | 传统咨询 | 本系统 |
|------|---------|--------|
| 交互方式 | 问卷式、模板化 | 对话式、引导式 |
| 分析深度 | 依赖顾问个人能力 | 基于理论框架+实时数据 |
| 响应速度 | 数周至数月 | 分钟级 |
| 成本 | 高（数万至数十万） | 低（订阅制或按次付费） |
| 可追溯性 | 低（依赖顾问记忆） | 高（全流程记录） |

### 2.2 核心功能模块

#### 模块一：初始化信息采集

**功能描述：** 用户首次进入系统时，采集基础信息用于后续分析。

**采集字段：**

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| 愿景(Vision) | 长文本 | 是 | 创始人认为企业成功时的样子 |
| 使命(Mission) | 长文本 | 是 | 为谁创造什么价值 |
| 价值观(Values) | 多行文本 | 是 | 核心行为准则（可多个） |
| 企业/项目名称 | 短文本 | 是 | - |
| 所属行业 | 下拉选择 | 是 | 预设选项+自定义 |
| 企业阶段 | 单选 | 是 | 0-1/1-10/10-N |
| 团队规模 | 数字 | 否 | - |
| 选定细分赛道 | 长文本 | 是 | 本次分析的核心赛道 |
| 补充说明 | 长文本 | 否 | 其他背景信息 |

**交互要求：**
- 支持分步填写，每步有清晰的引导文案
- 支持上传企业介绍文档（PDF/Word），自动提取关键信息
- 提交前展示信息摘要，允许修改

#### 模块二：多轮对话交互

**功能描述：** 核心交互界面，支持自然语言对话、文件上传、阶段流转。

**对话阶段设计：**

```
┌─────────────────────────────────────────────────────────────────┐
│ 阶段1：信息补充                                                  │
│ - 系统整理已采集信息，询问是否需要补充                            │
│ - 用户可自由补充或确认进入下一阶段                                │
│ - 触发条件：用户明确表示"信息完整"或"进入下一阶段"                │
├─────────────────────────────────────────────────────────────────┤
│ 阶段2：自由提问探讨                                              │
│ - 用户可就任何战略相关问题提问                                    │
│ - 系统基于已有信息进行回答，同时积累上下文                        │
│ - 用户可随时确认进入下一阶段                                      │
│ - 触发条件：用户明确表示"没有其他问题"或"进入下一阶段"            │
├─────────────────────────────────────────────────────────────────┤
│ 阶段3：预判采集                                                  │
│ - 系统反问：您对选定赛道十年后的预判是什么？                      │
│ - 用户输入预判内容                                               │
│ - 系统追问细节（市场规模、竞争格局、技术演进等）                  │
│ - 触发条件：用户明确表示"预判完整"或"开始分析"                    │
├─────────────────────────────────────────────────────────────────┤
│ 阶段4：分析报告生成                                              │
│ - 系统调用十年战略Agent进行分析                                  │
│ - 展示分析进度（搜索数据→整理论据→生成报告）                     │
│ - 生成完整报告，支持导出                                         │
│ - 用户可就报告内容继续提问                                       │
└─────────────────────────────────────────────────────────────────┘
```

**关键交互要求：**
- **非阻塞式流转：** 用户在任何阶段都可以继续提问，系统不强制中断
- **智能引导：** 系统在适当时机提示"您是否还有其他问题？"，但不阻止用户继续
- **上下文记忆：** 全程保持对话上下文，不丢失任何信息
- **文件解析：** 支持在对话中上传文件，自动解析并融入上下文

#### 模块三：十年战略分析Agent

**功能描述：** 核心分析引擎，基于用户预判生成正反论据和综合判断。

**输入：**
- 用户VMV信息
- 企业/项目基本情况
- 选定赛道
- 用户预判内容
- 对话过程中积累的上下文

**处理流程：**

```
┌─────────────────┐
│ 1. 解析预判内容  │ 提取关键假设、涉及维度
└────────┬────────┘
         ▼
┌─────────────────┐
│ 2. 搜索外部数据  │ 调用搜索API获取行业报告、政策文件、市场数据
└────────┬────────┘
         ▼
┌─────────────────┐
│ 3. 构建论据框架  │ 基于理论框架+搜索数据，构建正反论据
└────────┬────────┘
         ▼
┌─────────────────┐
│ 4. 生成综合判断  │ 整合论据，给出行动建议
└────────┬────────┘
         ▼
┌─────────────────┐
│ 5. 格式化输出    │ 生成结构化报告
└─────────────────┘
```

**输出结构：**

#### 模块四：报告导出

**功能描述：** 将分析报告导出为多种格式。

**支持格式：**
- Markdown (.md)
- PDF (.pdf)
- Word (.docx)

**导出要求：**
- 保留完整格式（标题层级、列表、表格）
- 包含生成时间、用户信息、会话ID
- 支持自定义文件名

### 2.3 非功能性需求

#### 性能要求

| 指标 | 目标值 |
|------|--------|
| 单次AI响应时间 | < 15秒（含搜索） |
| 报告生成时间 | < 60秒 |
| 文件上传限制 | 10MB |
| 并发用户数 | 10（Demo）/ 100（MVP） |

#### 可用性要求

| 指标 | 目标值 |
|------|--------|
| 系统可用性 | 99%（MVP阶段） |
| 单模型故障切换时间 | < 3秒 |
| 数据持久化 | 会话数据保留30天 |

#### 安全要求

- API密钥存储在.env文件，不提交代码库
- 用户数据加密存储
- 敏感信息（如企业财务数据）不记录日志

## 三、技术约束

### 3.1 技术选型原则

1. **简洁优先：** 优先选择成熟、文档完善的技术
2. **国产优先：** 大模型优先选择国产API
3. **可扩展：** 为五年、三年、一年Agent预留接口
4. **易维护：** 代码结构清晰，添加中文注释

### 3.2 技术栈建议

| 层级 | 技术选型 | 理由 |
|------|---------|------|
| 前端框架 | React + TypeScript | 生态成熟，组件丰富 |
| UI组件库 | Ant Design | 企业级UI，玻璃拟态可定制 |
| 后端框架 | FastAPI | 异步高性能，自动API文档 |
| Agent框架 | LangChain + LangGraph | 灵活的Agent编排 |
| 大模型API | 智谱GLM-4（主）+ 千问（备） | 国产，性能稳定 |
| 搜索API | Serper / Tavily | 支持中文搜索 |
| 文件解析 | PyMuPDF + python-docx | 成熟稳定 |
| 数据库 | SQLite（Demo）/ PostgreSQL（MVP） | 轻量级起步 |
| 缓存 | Redis（可选） | 会话状态管理 |

### 3.3 大模型配置策略

```python
MODEL_CONFIG = {
    "orchestrator": {  # 总控Agent
        "primary": "glm-4",      # 智谱GLM-4
        "fallback": "qwen-max",  # 千问Max
        "reason": "需要强上下文理解和调度能力"
    },
    "ten_year_agent": {  # 十年战略Agent
        "primary": "glm-4",
        "fallback": "qwen-max",
        "reason": "需要深度分析和逻辑推演能力"
    }
}
```

## 五、验收标准

### 5.1 功能验收

| 功能点 | 验收标准 | 优先级 |
|--------|---------|--------|
| 信息采集 | 所有必填字段可正常提交，数据正确存储 | P0 |
| 多轮对话 | 连续对话10轮以上，上下文不丢失 | P0 |
| 文件解析 | PDF/Word文件上传后，关键信息可被提取 | P1 |
| 十年分析 | 生成的报告包含正反论据各3条以上 | P0 |
| 信源标注 | 论据中的数据有明确的来源标注 | P1 |
| 报告导出 | MD/PDF/DOC三种格式均可正常导出 | P1 |
| UI风格 | 符合玻璃拟态风格，主流浏览器兼容 | P1 |

### 5.2 质量验收

| 指标 | 标准 |
|------|------|
| 代码规范 | 符合PEP8（Python）和ESLint（TypeScript）规范 |
| 注释覆盖 | 关键函数有中文注释，覆盖率>60% |
| 错误处理 | API调用失败有明确的错误提示和降级策略 |
| 日志记录 | 关键操作有日志，便于问题定位 |
